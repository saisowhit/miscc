What does it mean if my confidence interval includes zero?
If your confidence interval for a difference between groups includes zero, that means that if you run your experiment again you have a good chance of finding no difference between groups.

If your confidence interval for a correlation or regression includes zero, that means that if you run your experiment again there is a good chance of finding no correlation in your data.

In both of these cases, you will also find a high p-value when you run your statistical test, meaning that your results could have occurred under the null hypothesis of no relationship between variables or no difference between groups.



What is standard error?
The standard error of the mean, or simply standard error, indicates how different the population mean is likely to be from a sample mean. It tells you how much the sample mean would vary if you were to repeat a study using new samples from within a single population.


When should I remove an outlier from my dataset?
Itâ€™s best to remove outliers only when you have a sound reason for doing so.

Some outliers represent natural variations in the population, and they should be left as is in your dataset. These are called true outliers.

Other outliers are problematic and should be removed because they represent measurement errors, data entry or processing errors, or poor sampling.

When testing hundreds of hypotheses, there are several things to consider, including:
Multiple comparisons problem
The probability of making an error increases as the number of hypothesis tests increases. This is known as the multiple comparisons problem, multiplicity, or the look-elsewhere effect.
Computational efficiency
Techniques like group testing and hierarchical testing can help reduce the computational burden.
Bayesian approaches
Bayesian methods can incorporate prior information and provide direct probability statements about the hypotheses.
False discovery rate (FDR)
In some situations, it's not possible to control for the family-wise error rate, so instead the false discovery rate is controlled. This allows for a certain proportion of hypotheses to be false positives. The Benjamini-Hochberg correction is a common method for controlling the false discovery rate.
Validation tests
Validation tests based on multiple testing correction, such as Bonferroni and Benjamini and Hochberg, can help correct for the occurrence of false positives.
Permutation tests
Resampling tests, such as permutation tests, can help correct for the occurrence of false positives.
There are two types of multiple hypothesis testing:
Global hypothesis testing: All null hypotheses are tested simultaneously.
Multiple testing: Each null hypothesis is tested separately.

When testing hundreds of hypotheses, there are several things to consider, including:
Multiple comparisons problem
The probability of making an error increases as the number of hypothesis tests increases. This is known as the multiple comparisons problem, multiplicity, or the look-elsewhere effect.
Computational efficiency
Techniques like group testing and hierarchical testing can help reduce the computational burden.
Bayesian approaches
Bayesian methods can incorporate prior information and provide direct probability statements about the hypotheses.
False discovery rate (FDR)
In some situations, it's not possible to control for the family-wise error rate, so instead the false discovery rate is controlled. This allows for a certain proportion of hypotheses to be false positives. The Benjamini-Hochberg correction is a common method for controlling the false discovery rate.
Validation tests
Validation tests based on multiple testing correction, such as Bonferroni and Benjamini and Hochberg, can help correct for the occurrence of false positives.
Permutation tests
Resampling tests, such as permutation tests, can help correct for the occurrence of false positives.
There are two types of multiple hypothesis testing:
Global hypothesis testing: All null hypotheses are tested simultaneously.
Multiple testing: Each null hypothesis is tested separately.

When testing hundreds of hypotheses, there are several things to consider, including:
Multiple comparisons problem
The probability of making an error increases as the number of hypothesis tests increases. This is known as the multiple comparisons problem, multiplicity, or the look-elsewhere effect.
Computational efficiency
Techniques like group testing and hierarchical testing can help reduce the computational burden.
Bayesian approaches
Bayesian methods can incorporate prior information and provide direct probability statements about the hypotheses.
False discovery rate (FDR)
In some situations, it's not possible to control for the family-wise error rate, so instead the false discovery rate is controlled. This allows for a certain proportion of hypotheses to be false positives. The Benjamini-Hochberg correction is a common method for controlling the false discovery rate.
Validation tests
Validation tests based on multiple testing correction, such as Bonferroni and Benjamini and Hochberg, can help correct for the occurrence of false positives.
Permutation tests
Resampling tests, such as permutation tests, can help correct for the occurrence of false positives.
There are two types of multiple hypothesis testing:
Global hypothesis testing: All null hypotheses are tested simultaneously.
Multiple testing: Each null hypothesis is tested separately.
